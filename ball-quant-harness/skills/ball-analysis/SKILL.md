# Ball-Analysis Controller —— 世界杯体彩 LLM 决策系统

## 0. 这是一套环环相扣的因果系统（先读）
`背景 + 要求 → 工作流 → 数据路由 → 提示词 → 输出`。任何一环错，输出就垮。
本 skill 是**控制器**：决定**何时给哪个 agent、什么证据、什么数据、用什么提示词**；最终决策由 **LLM（你）对全部原始数据做推理**得出。

**五条铁律：**
1. **采集 agent 只搬原始数据**——禁止算 edge / 隐含概率 / Kelly、禁止排序 / 选"最优" / 下结论 / 给数字加"强弱好坏"形容词。任何压缩都是提前替人做决定，污染信号。
2. **全部原始数据喂给推理层**，绝不预筛、预总结、预排序。
3. **决策全由 LLM 按 §4 框架推理**得出，不是代码算的。
4. **只做世界杯。**
5. **以 球队实力(KG) + Polymarket 真金 + 因果 为主导；赔率 / edge 只是校验，绝不被高赔率牵着走。盈亏比永远不脱离胜率/概率/直觉；用全套玩法+串关；每单必配防爆冷。**

## 1. 背景不变量（Background —— 放这里，别塞进任务）
- **体彩 5 玩法（全部 90 分钟常规时间结算，不含加时点球）**：胜平负 · 让球胜平负 · 比分 · 总进球(0–7+) · 半全场。BTTS / 单队进球 / 大小球"线" **不是体彩玩法**（仅用于读 Polymarket 倾向 + 给总进球档锚定）。
- **投注形态（全用，别只押一种玩法）**：单关 · 串关(2串1 … 8串1) · **混合过关**(不同玩法 / 不同场次混合成串)。让球/总进球/比分/半全场都可单可串。
- **倍投**：单注 ¥2；单张票 ≤99 倍（≈¥198），大额拆多张；串关奖金封顶 2-3串20万 / 4-5串50万 / 6+串100万。单关只在指定玩法/场次开，临场在 sporttery 核。
- **edge 口径**：`edge = P_polymarket × 体彩赔率 − 1`。这是**校验尺，不是驱动**——用来看体彩哪里偏离了你的科学判断，不能用它代替剧本。
- **玩法精确定义（最易错，必须先写清再下注）**：`总进球=N` = 全场**两队进球总数**恰为 N，覆盖 N-0 / (N-1)-1 / … / 0-N **多个比分（含平局、含任一方赢）**——**绝不等于"某队 N-0 赢"**。`让球X·主/平/客` = 让球后 90 分钟结果(主=让后仍胜/平=让后打平/客=让后客胜)，逐档写清条件。`比分` = 精确比分。论证一注前，先用一句话写出"这注到底在什么结果下赢"。
- **盈亏平衡概率（每注必算，串关尤其）**：单注 = `1/赔率`；**串关 = `1/(各腿赔率连乘)`**，串关命中概率 = 各腿命中概率**连乘**。一注只有 `命中概率 > 盈亏平衡概率`（即 edge>0）才"占优"；**低于盈亏平衡就是 −EV，必须明标，禁止用"胜率高"包装 −EV 注**。
- **信心分级**：胜平负·让球 = 市场锚定（直接对 Polymarket moneyline/handicap 去水价）最可信；总进球档可由 Polymarket 大小球线**差分**锚定 `P(=n)=P(over n−0.5)−P(over n+0.5)`；比分/半全场多为网格外推（软，需剧本+锚定双支撑）。
- **风险姿态（用户要求）**：**满买**（不留干火药、不"建议别买"）；**分层组合**(背骨/盈亏比/价值/防守)；**盈亏比 × 胜率 平衡 + 强制防爆冷**。
- **铁律6 · 必须交付（最重要的态度）**：用户给了预算/要求，就**必须果断给出最优买法**，**绝不以"−EV / 建议别买"搪塞**。体彩有抽水→真 +EV 罕见；在"既然要买"前提下，**融汇贯通**(真金概率+球队实力+因果+战术+直觉)给出**风险调整后最佳 + 有上限**的票，像懂球的哥们用真钱给推荐：**先给票、再用一句话提风险**，不本末倒置。规则要严（票面/盈亏平衡不许错），但姿态要进取、有温度——目标是**让用户赢**。

## 1.5 体彩规则 × Polymarket 映射 硬约束（最易错——违反即作废重算）
**体彩让球胜平负 ≠ Polymarket 亚盘。绝不可把 −1.5/+1.5 直接当体彩让球。**
- **让球胜平负**：让球线 N 永远从**主队**视角（主队让 N 球；c500 `data-rangqiu` 负=主让、正=主受让）。三个结果：
  - **让胜** = 主队净胜 > |让出的球|（主让 1 → 主赢 **2+**）。
  - **让平** = 主队净胜**恰好等于**让球数（体彩的"走水"是这个独立选项）。
  - **让负** = 主队净胜不够 / 平 / 负。
  - 例：阿根廷(主)让1 → 让胜=阿根廷赢 **2+**；让平=阿根廷**恰好赢1**；让负=阿根廷赢≤0。
  - 例：伊拉克(主)+2、挪威(客) → **客胜 = 挪威赢 3+**（**不是 2+**！）；让平=挪威**恰好赢2**；主胜=挪威赢≤1/平/伊拉克赢。
- **体彩没有亚洲盘 ±0.5/±1.5 走水**。Polymarket 的 `−1.5`(赢2+)/`+1.5`(不输2+) **只能当概率参考**，落体彩票面**必须**翻成整数让球的 让胜/让平/让负。"客队+1 走水"不是合法票面 → 写成"主队-1 让平/让负"。
- **总进球数=N** = 全场**两队总进球**恰为 N（覆盖 N-0 / (N-1)-1 / … / 0-N，**含平局、含任一方赢**）。要"某队 N-0 赢"→ 用**比分**，不用总进球。
- **盈亏平衡**：单 = 1/赔率；**串 = 1/(各腿赔率连乘)**，命中 = 各腿连乘。命中 < 盈亏平衡 = −EV，**必标，禁止用"胜率高"包装**。
- **禁止玩法混淆**：别把多个不同玩法都说成"某队赢→全受益"。每注写清**独立中奖条件**，再写它们**真正同时中**的联合条件（如只有"阿根廷 2-0"才同时命中 让胜+总进球2+比分2-0）。
- **事实必须有源**：球员/伤停/进球数引用要核（KG enrich 里的数字可能未核，用前 WebSearch 验，如 Haaland 世预赛16球而非"资格赛37球"）。

## 1.6 三层概率分离 + 首轮闷局先验（全平局日复盘铁律 — 最高优先）
**第一原则：先判"能不能穿盘"，再判"谁更强"。强 ⇏ 穿盘 ⇏ 比分。**
- **三层概率，单调递减，必须分别取数，严禁用胜率的信心去套深盘/比分**：
  - **L1 胜负** P(胜/平/负) ← 强队实力 + Poly **moneyline**。强队实力只支撑这一层（"不败/胜率高"）。
  - **L2 穿盘** P(让N 穿盘) ← Poly **spread/handicap** 市场（或比分网格边际）。**与 L1 是两个独立的数**（实证：ARG 胜 70.5% 但 ARG-1.5 仅 43.5%；西班牙 胜 91% 但 -2.5 仅 ~42%）。
  - **L3 比分** P(精确比分) ← Poly **correct-score** 市场。方差最高，**永不主仓**。
  - 不变式：`P(win) > P(cover −1) > P(cover −2) > P(exact)`。越深越低，**别把"强队"映射成"赢2球/2-0"**。
- **首轮无可靠方向先验（06-15全平 vs 06-17全大胜 → 双向证伪）**：别给首轮叠"闷局/受让"方向先验——两个方向各 n=1 = **噪声**。**信 Poly 校准的胜负排序**（两天都排对了），别叠自己的剧情。唯一微调：小组赛**净胜球动机**让强队倾向"赢得舒服" → **轻微偏向热门赢/穿盘，绝不默认 fade 热门**。
- **−EV ≠ 价值（铁律·屡犯，最早 ¥1000 表就栽过、06-17 又犯）**：edge≤0 的腿**禁止**叫"价值/value"——它就是"近公平的博冷 / −EV"，老实标。**只有 edge>0 才配叫价值。** <35% 且 edge≤0 的受让/冷门腿，除非有强因果，**不买**。
- **爆点缺失降权**：依赖边路爆破/个人创造的队（Yamal/Nico 类），核心爆点缺阵/替补 → **对"穿深盘/破密防"概率显著降权**（不止降胜率——密防更容易守住）。
- **动态降级（早盘/live）**：开赛 20–25 分钟仍 0-0、热门无持续高质量机会 → 2-0/深让票风险立即上调、快速降级。
- **串关纪律（06-17修正）**：**热门胜的短赔串 = 合法的"命中率/爽"结构**（实证：奥胜×阿胜×法胜 三连全中）——可买，但标清"为命中买、非 +EV"，别叫价值。受让/冷门腿**只在 edge>0 时**才进串，绝不因"赔率高"当价值串。
- **实证·双向（关键 — 别赌方向）**：**06-15** 四场全平 → 受让/让负全红；**06-17** 三热门 **3-1/3-0/3-1 全打穿** → 热门胜/穿盘全红、受让全挂。**两天完全相反 = 没有"首轮方向规律"，唯一稳的是 Poly 胜负排序两天都对** → 信排序、热门胜做命中骨架，别赌闷/爆方向。

## 1.7 结构有效性闸门（Codex 对抗评审 2026-06 — 比统计精度更底层，不过这关 edge 全是假的）
**核心反思**：`edge = P_poly × 体彩赔率 − 1` 可能**系统性错而非噪声错** = **basis risk**：拿"一个合约的去水价"比"另一个合约的固定赔率"，默认二者同一随机变量。任一结构错位 → 算的是**错随机变量的 EV** → 持续显示假正 edge、实则 −EV。先过下面闸门，再谈 edge：

- **T0 存亡（没过 = 一切归零）**：
  - **结算等价**：每个 Poly↔体彩 配对必须同一事件同一口径（90分钟 vs 全场/加时/点球；改期/腰斩/中立场/void；结果划分）。不等价 → 不下注。（呼应未修的日期错配 bug。）
  - **basis risk = 默认零假设**：验证前，`P_poly` 只是意见源，不是 edge。
  - **验证用校准+ROI，不是 CLV**（Codex-5.5 纠正）：体彩是固定赔率池、不"收盘"，"打赢收盘线"只证明开赛前出了票。正确指标 = frozen 概率**校准**(Brier/log-loss/校准回归 α≈0、β∈[0.8,1.2]、ECE≤2.5pp、Brier skill>baseline) + **税后实现 ROI**(match-clustered 方差) + 对 baseline 的 **LLR**。CLV 仅对**独立 sharp 参考**有效。没样本前**纸面跑，不放大真金**。
- **T1 结构（edge 数学的前提）**：
  - **比赛内 = 一个联合状态空间 + 一个敞口预算**（Codex-5.5 纠正"一注"过强）：5 玩法是同一进球路径的投影；不禁多腿，而是**全部腿过同一个 state-contingent 组合优化器**，第二腿仅当**协方差+最坏状态后边际 log-growth 仍为正**才给仓；敞口在"场"层封顶（每场 ≤0.5% bankroll、比分+半全场合计 ≤0.1%）。
  - **Poly 可能是软的一方**（Codex-5.5，最深）：冷门/薄盘（比分/半全场/总进球/低流动 derivative）Poly 比体彩机构定价更散、易被单笔带偏 → "体彩 stale"实为**在打更 sharp 的体彩线**。**edge 只现于薄 Poly 盘 = 默认可疑**；真 edge 在**流动主盘**(1X2/主让/主大小)。必过 **oracle 质量门**(spread/深度/成交数/contract age/cross-market 一致/单笔敏感)。**别再猎冷门软点。**
  - **Poly 用可执行价 + 新鲜度**：bid/ask 边界 + 深度 + recency；mid-price ≠ fair。edge 须扛过**最坏 fair 区间 + 多种 devig（proportional/Shin/power）**。
  - **选择性收缩 + 下界阈值**：扫 N 盘挑最大 edge → 按搜索宽度 + 噪声收缩（winner's-curse haircut）。用**下界概率**：主盘 EV_LB≥+3%、比分/半全场≥+8~12%、薄盘≥+15%。
- **T2 下注**：分数 Kelly on 收缩后验 edge（η=0.10 起）；**默认不下注**；每场敞口硬上限；**杀"满买"**。体彩抽水不均匀——但软点在**流动主盘里体彩堆公众水位、而 Poly 仍可靠**之处，**不在冷门盘**。
- **T3 精修**：三源（Poly spread / 体彩比分总进球隐含 / Poisson 网格）**先对齐再看残差**，只有残差是信号；跨场 regime 因子（低优先）。
- **样本量现实（决定性）**：证 5% edge @2.0 需 **~2500 注**、3% 需 ~6900、@4.0 的 5% 需 ~7700；现实下注 ~0 → **统计上未证明**。这是**预测+纪律框架，不是已证提款机**；"edge"当假设，先起 logging harness（每场每盘 1 个 pre-registered 快照）攒样本，校准达标前不放大。
- **别过拟合 n=1（已两次踩坑）**：先按 06-15(全平) 立"fade 热门"先验，又被 06-17(全大胜) 证伪——**单日结果绝不立规律**。手估概率（Poly 没直报的盘）= fabricated precision，标 `[手估]` 不当 edge。

## 1.8 快速执行结构（慢的根因 = 串行重复拉数 → 改成"一次过"）
- **Step 1 · 一次性全量 sweep（1 个调用，杜绝重复拉）**：cron 已在缓存 Poly，直接读最新缓存。per match 一次性吐出 **Poly 全角度**（1X2 / 让球各档 / 大小各档 / 比分 / BTTS / 半场 / 球队总进球，带去水概率 + 流动性）+ **体彩 5 玩法赔率 + 单关/过关/未开售 flags** + KG 强度/伤停。**不 live 重抓、不分次拉。**（提速器 = 建 `ballq bundle --date` 一条命令吐这个 raw bundle，秒回。）
- **Step 2 · 多角度读剧本（LLM，无 agent，秒级）**：把一场的多个真金盘口融成**一句剧本**——`1X2+平局资金`=谁主导/平局活不活；`让球各档`=净胜分布(碾压/小胜/闷平)；`大小各档`=进球环境;`BTTS`=单边vs对攻;`比分modal+半场`=最可能比分+上半节奏。**权重：流动主盘(1X2/让/大小)定剧本骨架，薄盘(比分/半场)只上色**（薄盘 Poly 软，不全信——§1.7）。
- **Step 3 · 均衡 RR 买（核心 + 穿插博）**：**核心(低风险)** = 剧本最高确信 × 流动主盘 → 吃胜率 floor；**穿插博(高RR)** = 跟剧本一致的小注高赔 → 博上限、小仓封顶；core 给地板 + spec 给上限按 RR 配比，每场敞口封顶(§1.7)。
- **铁律**：一次 sweep 拿全 → 剩下全是 LLM 推理，**0 次再拉数据**。再分次拉 = 结构没优化好。

## 1.9 竞彩规矩（全文见 `JCZQ_RULES.md` — 买票前必读 flag，今天栽在这）
- **串关=默认；单关=体彩按"场×玩法"逐一开的特权**：买前必查每玩法的 `data-subactive` **单关(dg)/过关(gg)** flag，**绝不假设**。`*dg:1`可单买；`*dg:0,gg:1`只能进串；`*dg:0,gg:0`没开售。
- **胜平负/让球单关稀缺**（仅大赛指定场次）；**比分/总进球/半全场常规单关**；**同场可分裂资格**（比分能单、胜平负只能串，正常）。
- **同一场不同玩法不能混进同一注串** → 一场押多玩法只能**各自单关分票**（这是 §1.7"比赛内=一注"的**规则级强制**，不是建议）。
- 串腿上限：胜平负/让球 **8**、总进球 **6**、比分/半全场 **4**；混合串取最低（木桶）。
- 奖金封顶：单关 **500万** / 过关 2-3场 20万 / 4-5场 50万 / 6+场 100万；倍投 ≤**99倍** & ≤**¥2万**/票；¥2/注。
- 结算 **90分钟**(不含加时/点球)；改期→单关退票、串关**剔腿降档结算**（不退全本）。

## 1.10 资金分层结构 A/B/C（用户框架 + EV 纠正 — §1.7 gates 的用户向前端）
**本质（必须先认）**：这是 **VARIANCE/生存系统，不是 EV 系统**。substrate 是 −EV（体彩抽水），A 层也 −EV；分层只决定"**怎么亏**"(慢/低波动)，不决定"亏不亏"。**目标 = 生存 + 娱乐(低 ruin)，非增长——除非 B 层验出真 edge。没真 edge 的夜晚，EV-最优下注额 = 0。**
- **A层·生存兜底（50-70% × 小总额）**：高概率(Poly≥60%) + 不串死 + 不深盘 + 不靠比分。作用=**低波动兜底，仍 −EV**。选腿挑 **edge 最接近 0 的高概率腿**（最不坑的水），别压 @1.1x 那种最深的水。
- **B层·唯一 +EV 潜力（0-30%·条件注资）**：**只有 edge>0（收缩后、过 oracle 质量门）才注资；没验出 → B层 = ¥0**。size = 分数 Kelly on shrunk edge。"验出 edge"需 CLV/校准样本，非拍脑袋。**这是系统能否真盈利的唯一开关。**
- **C层·尾部娱乐（0-10%·可删）**：≤10% 事件、比分/深盘，必须**可全损**、严格封顶。可删层非必需。
- **死亡路径闸（铁律）**：整套结构 **P(全亏=¥0) < 10%**；超阈值自动砍 C / 拆串，禁 all-or-nothing。
- **进层 filter（Poly 概率 × edge-sign 双门）**：Poly<25% 禁入(除 C 且有强因果)；25-45%→只 C；45-60%→最多 B；≥60%→可 A。**叠加：A/B 层每条腿 edge 必须过阈值**（别把生存钱压在最坑短热门上）。
- **总额纪律**：总额随"有没有 B层 edge"伸缩——**没 edge 的夜晚压到娱乐级**。分层管波动，不管 −EV；别因"有分层结构"就满买。

## 4. 推理层框架（LLM 分析师协议 —— 拿到全部 <evidence> 后这样 think）
在 `<thinking>` 里按序（**实力与真金为先，赔率为辅**）：
- **Step1 引用原始证据**：逐场摘出 Polymarket 1X2/让球/大小球分布、KG 实力(strength_win/rank)/远期/伤停/战术、体彩各玩法赔率。对原文推，不对印象推。
- **Step2 定剧本（实力+真金+因果为主，此步不看体彩赔率）**：用 **KG 实力 + Polymarket 真金概率 + 因果(远期/伤停/战术)** 判这场**最可能怎么走**——谁赢、赢多少、节奏高低。纯科学读盘。"现在单场线弱但远期强→后程发力""强队让球大胜 vs 小组赛保守"等因果在此用。
- **Step3 估冷门（防爆冷）**：独立给每场的冷门概率 + 触发条件(反击型弱旅、强队伤停、揭幕保守)，标出**最可能翻车的那场**——它是防守票的目标。
- **Step4 赔率校验（edge 只是尺，不是驱动）**：每个候选（**含串关：命中概率=各腿连乘、赔率=各腿连乘**）都必须并列写出 `命中概率` 与 `盈亏平衡概率=1/赔率`，再算 `edge=P×赔率−1`。**命中概率 < 盈亏平衡 → −EV，老实标 −EV，禁止用"胜率高/剧本好"伪装成占优注**。edge 用于选表达方式/排优先级，不代替剧本。**高赔率≠好注；高胜率若低于盈亏平衡线也是坏注。**
- **Step5 全工具选型**：把剧本翻译成体彩玩法——胜平负 / 让球(强弱差大→用让球放大盈亏比) / 总进球 / 比分 / 半全场，单关或**混合过关串**。同一剧本优先选 **胜率×盈亏比** 更优的表达。
- **Step6 分层组合（盈亏比 × 胜率 × 防爆冷）**：
  - **背骨(高胜率)**：最稳的强队赢，尊重实力、吃概率，权重最大。
  - **盈亏比(搏碾压)**：用让球/串关把"实力碾压"放大成高赔，权重中。
  - **价值单(+EV)**：体彩明显偏离科学判断的那注（若有）。
  - **防守票(防爆冷)**：盖 Step3 标的最可能冷门，权重小但**必配**。
  - **倍投把预算打满**(¥2×倍, >¥198 拆票)，不留干火药。
- **Step7 裁决**：给确定的分层买单，每注标 **命中概率 + 盈亏比 + 角色**。不列利弊不决。

## 5. 输出契约（分层买单）
- **表**：层(背骨/盈亏比/价值/防守) | 比赛 | 玩法(单关/串关) | 选择 | 体彩赔率 | P(命中) | edge | 金额(倍投) | 命中回报。
- **逐注理由**：实力 + 真金 + 因果（为主）→ 剧本 → 为什么这个玩法/这个价（赔率校验）。
- **防爆冷覆盖**：明说防守票盖哪场冷门、触发时怎么兜。
- 总投入 = 预算（**满**）| 整体胜率画像 | 最好/最坏情形 | 一句"盈亏比 × 胜率"总览。
- **诚实标注**：快照赔率会变（下注前核当前）；软玩法信心低；**若全场体彩定价有效（无 +EV），明说"主要是付水位买实力剧本"，不假装有肉**。
