# Ball-Analysis Controller —— 世界杯体彩 LLM 决策系统

## 0. 五条铁律（先读）
1. **采集 agent 只搬原始数据** — 禁止算 edge / Kelly / 结论 / 形容词。
2. **全量原始数据喂推理层** — 不预筛、不预总结。
3. **决策全由 LLM 按 §4 框架推理** — 不是代码算的。
4. **只做世界杯。**
5. **实力(KG) + Polymarket 真金 + 因果 为主导；赔率 / edge 只是校验，不被高赔率牵着走。**

## 1. 背景不变量

- **体彩 5 玩法（全部 90 分钟结算，不含加时点球）**：胜平负 · 让球胜平负 · 比分 · 总进球(0–7+) · 半全场。BTTS/单队进球/大小球"线"不是体彩玩法（仅用于读 Polymarket 倾向）。
- **投注形态**：单关 · 串关(2串1…8串1) · 混合过关。让球/总进球/比分/半全场都可单可串。倍投：单注¥2；单张票 ≤99 倍；串关封顶 2-3串20万/4-5串50万/6+串100万。
- **edge 口径**：`edge = P_polymarket × 体彩赔率 − 1`。校验尺，不是驱动——用来看体彩哪里偏离科学判断，不能代替剧本。
- **玩法精确定义（最易错）**：`总进球=N`=全场两队总进球恰为N（含平局含任方赢，绝非"某队N-0"）；`让球X·让胜/让平/让负`=让球后90分钟；比分=精确比分。**论证前先一句话写清"这注在什么结果下赢"**；每注写清独立中奖条件+多注同时中的联合条件；事实引用必须有源，手估标`[手估]`。
- **盈亏平衡**：单=1/赔率；串=1/(各腿赔率连乘)；命中=各腿连乘。命中<盈亏平衡=−EV，**必标，禁"胜率高"包装。**
- **信心分级**：胜平负·让球=主盘最可信；总进球档可差分锚定；比分/半全场=软，需剧本+双支撑。

## 1.5 体彩规则 × Polymarket 映射 硬约束

- **体彩让球胜平负 ≠ Polymarket 亚盘。** 让球线N从主队视角：让胜=主净胜>让球数；让平=恰等；让负=不够/平/负。
  - 例：阿根廷(主)让1 → 让胜=赢2+；让平=赢恰好1；让负=赢≤0。
  - 例：伊拉克(主)+2，挪威客 → 让胜=挪威赢≤1/平/伊赢；让平=挪威赢**恰好2**；让负=挪威**赢3+**。
- **体彩没有亚盘±0.5/±1.5走水**。Polymarket `−1.5`/`+1.5` 只当概率参考，落票面必须翻成整数让球。

## 1.6 三层概率分离

**第一原则：先判"能不能穿盘"，再判"谁更强"。强队 ⇏ 穿盘 ⇏ 比分。**
- **L1 胜负** P(胜/平/负) ← 实力+Poly moneyline。强队只支撑此层。
- **L2 穿盘** P(让N穿盘) ← Poly spread/handicap。**与L1是两个独立数**（ARG胜70.5% vs ARG-1.5仅43.5%）。
- **L3 比分** P(精确) ← Poly correct-score。方差最高，**永不主仓**。
- 不变式：`P(win) > P(cover−1) > P(cover−2) > P(exact)`。
- **无可靠方向先验（铁律）**：别给首轮叠"闷局/受让"方向先验——信 Poly 校准的胜负排序（两天都排对），不赌闷/爆方向。唯一微调：净胜球动机让强队倾向"赢得舒服"→轻微偏向热门穿盘，不默认 fade 热门。
- **−EV ≠ 价值（铁律·屡犯）**：edge≤0 禁叫"价值"；<35% 且 edge≤0 的冷门腿，除非强因果，不买。
- **串关纪律**：热门胜短赔串=合法"命中率"结构，但标"为命中买、非+EV"。受让/冷门腿只在 edge>0 时进串。

## 1.7 结构有效性闸门（不过这关 edge 全是假的）

- **T0 存亡（没过=归零）**：结算等价（同一事件同口径：90分钟 vs 全场/加时；不等价→不下注）；basis risk=默认零假设（验证前 P_poly 只是意见源）；验证指标=Brier/log-loss/ECE≤2.5pp + 税后ROI(match-clustered)；CLV仅对独立sharp参考有效；没样本前纸面跑。
- **T1 结构**：
  - **within-match = one-bet（新增）**：同一场内两腿绝不组串——同场各腿共享同一进球路径，相关性高，串联翻转 edge 符号。**案例：刚果金+2（P=0.485，+3.3%）× 葡萄牙胜（P=0.765）→ 联合条件="葡萄牙恰好赢1球"（P≈0.25，EV≈−40%）；1-1使串失败，单注刚果金+2命中。** 同场多玩法只能各自单关分票（亦是竞彩规则强制，§1.9）。
  - **仅过关锁出真实 edge → PASS（新增）**：唯一 +EV 腿若 dg:0,gg:1（仅过关），所有 −EV 搭档拖低联合 EV → 诚实动作=PASS，不强串 −EV。**案例：2026-06-17 刚果金+2唯一+EV但仅过关；搭档均−EV → PASS；赛果1-1，孤腿中，无处落单。**
  - **比赛内全部腿过组合优化器**；每场敞口 ≤0.5% bankroll，比分+半全场合计 ≤0.1%。
  - **Poly 薄盘可疑**：比分/半全场/总进球 Poly 流动低，易被单笔带偏 → edge 只现于薄 Poly 盘=默认可疑；真 edge 在流动主盘。必过 **oracle 质量门**（spread/深度/成交数/recency/cross-market一致）。Poly 用可执行价：bid/ask+深度+recency；mid≠fair；edge须扛最坏fair区间+多种devig。
  - **选择性收缩+下界阈值**：主盘 EV_LB≥+3%；比分/半全场≥+8~12%；薄盘≥+15%。
- **T2 下注**：分数 Kelly（η=0.10起）；**默认不下注**；每场敞口硬上限。
- **T3 精修**：三源（Poly spread/体彩比分总进球隐含/Poisson网格）对齐后只有残差是信号。样本量现实：证5% edge @2.0需~2500注；先起 logging harness 攒样本，校准达标前不放大。

## 1.8 默认不下注 / 四层资金结构 45/30/15/10（强制默认）

**Default = NO-BET。NEVER "直接给票"——任何出票前必须先完成资金层设计（§1.10 五步模板）。**

**A/B/C/D 四层硬比例**（每场赛事日默认；无 edge 整场=PASS，层比例=¥0）：

| 层 | 占比 | 定位 | 进入条件 | 玩法 | 票数 |
|---|---|---|---|---|---|
| **A** 平胜负/不败 | **45%** | 别死 | Poly单边>55% OR 平>25%&市场压低比分 OR 不败>75% | 单关胜平负 / 低风险2串1 | 2-4票 |
| **B** 剧情爆发串 | **30%** | 舒服赚 | ≥2 Poly盘口支撑**同一剧情** | 胜平负×总进球 / 胜平负×让球 / 让球×总进球 | 3-5票 |
| **C** 比分单关 | **15%** | 抓最像比分 | Poly最支持的单场比分；prob>5%（最好>8%）且与胜负/让球/总进球/BTTS一致 | 单场比分单关 | 3-6票，¥2-8/票 |
| **D** 比分串高爆 | **10%** | 高爆不污染主仓 | each比分>5%；剧情一致；返奖≥5×总预算 | 2串（现实命中）>3串（彩票） | 2-4票 |

> **D层比分一致性铁律（严禁替换）**：D层比分串每条腿 **必须 = 该场 Poly/C层最高概率的比分本身**；高赔率只能来自「串顶分相乘」，**绝不为赔率换成命中率更低的邻近分**；C 与 D 必须比分自洽（C 押 3-0 → D 只能串 3-0）。押邻近分（如3-1/2-1代替3-0/2-0）= 直接放弃已知最高概率路径，历史教训：赛果恰好落在C层正确比分而D层因替换全损约¥750/¥100预算。

**预算换算**：¥100→44/30/16/10 · ¥200→90/60/30/20 · ¥400→180/120/60/40

**微调规则**（偏离默认须显式声明理由）：
- 主线极强 → A40/B35
- 冷门概率高 → A55/B20
- 纯高爆模式 → A30/B30/C20/D20（彩票模式，全亏率显著升，须标注）

**本质（保留铁律）**：这是 **VARIANCE/生存系统，不是 EV 系统**。A层也可能−EV；分层决定"怎么亏（慢/低波动/保留上行）"，不决定"亏不亏"。**没真edge的夜晚，EV最优额=0，总投入压娱乐级。** 多数夜晚=小注或PASS。
- **不追高赔（A层铁律）**：A层不进高赔腿，不靠比分，不串死。
- **死亡路径闸（铁律）**：整套结构 P(全亏=¥0) < 10%；超阈值自动砍D/拆串，禁 all-or-nothing。
- **进层 filter**：Poly<25%禁入（除C/D且有强因果）；−EV ≠ 价值（铁律·屡犯），禁"胜率高"包装。

## 1.10 默认输出模板（5步）+ 出票前4过滤器

**每个赛事日，必须按序走完5步，才能出票。**

**Step ①** Poly盘口解构：逐场列 胜平负 / 让球 / 总进球 / BTTS / 球队进球 / 半全场 / 比分簇 原始概率（带深度/recency标注）。

**Step ②** 剧情分层：基于①，给出本场主线剧情（主胜/平冷门/进球数/比分方向）及触发条件；标"最可能翻车那场"（冷门风险）。

**Step ③** 资金分配 45/30/15/10：按§1.8四层公式算出本日各层金额；微调须写理由；无edge的层→¥0，不强填。

**Step ④** 分层出票清单（A/B/C/D）：每票：层 | 比赛 | 玩法 | 选择 | 体彩赔率 | P(命中) | edge | 金额 | 命中回报 | 中奖条件（一句话，what result wins this bet）。

**Step ⑤** 审计7问（全过才出票）：
1. 主线剧情是否支撑A层？
2. B层≥2个Poly盘口支持同一剧情？
3. A层有无防死路径（冷门爆发时是否至少有一腿命中）？
4. 有无垃圾票（−EV且无剧本支撑）？
5. 全部4层同时落空概率（全亏P）< 10%？
6. D层返奖 ≥ 5× 总预算？
7. 最小回收路径（A层全中）≥ 多少？

**出票前4过滤器（顺序检查，任一否→修改对应层/票）**：
1. Poly主线盘口（胜平负/让球/总进球）全部一致？矛盾不重仓。
2. C/D层比分选择与胜负/让球/总进球盘口方向一致？
3. B层让球方向与总进球档对齐？（让大分赢的剧情 vs 低比分剧情 = 矛盾）
4. D层各串腿联合返奖 ≥ 5× 总预算？否则降级为C层单关或取消。
5. **D票比分核验**：每条腿是否 = 该场 Poly 最高概率比分 **且** 与 C层选择完全一致？有任一腿为邻近/替换比分 → **不出**（回§1.8 D层铁律）。

## 1.9 竞彩规矩（全文见 `JCZQ_RULES.md` — 买票前必读）

- **串关=默认；单关=逐一开的特权**：买前必查每玩法的 `data-subactive` **单关(dg)/过关(gg)** flag。`dg:1`可单；`dg:0,gg:1`只能进串；`dg:0,gg:0`没开售。
- **胜平负/让球单关稀缺**（仅大赛指定场次）；比分/总进球/半全场常规单关。
- **同一场不同玩法不能混进同一注串** → 各自单关分票（规则级强制，§1.7 T1 within-match 的规则根源）。
- 串腿上限：胜平负/让球8、总进球6、比分/半全场4；混合取最低。
- 结算 90分钟（不含加时/点球）；改期→单关退票、串关剔腿降档结算。

## 2. 控制器工作流

触发后：
1. 取当日 WC 比赛清单：`ballq poly-schedule --date <today> --timezone Asia/Shanghai`，只留世界杯。
2. **跑一条 `ballq bundle --date <d>`** → LLM 直接读 bundle → LLM 按 **§1.10 五步模板**产出分析（①Poly盘口解构 ②剧情分层 ③资金分配A/B/C/D 45/30/15/10 ④分层出票清单 ⑤审计7问 + 4过滤器 → 小注/PASS）。**NEVER "直接给票"——§1.10五步模板必须走完才出票。**
3. **`recommend` 命令**：legacy 代码候选 / 粗筛 — **不是产品答案**。产品答案 = LLM 对 bundle 的分析。

## 3. 采集 agent 提示词模板（raw only）

```
<role>你是数据检索 agent，唯一职责=按源取原始数据原样贴出；禁止算任何派生量/结论/形容词。</role>
<task>取 [比赛/盘口]：1)[字段A] 2)[字段B] 3)[字段C]。逐项贴 label+原始值+源+时间。</task>
<output_format>只回：<raw_data><item label="…">值 单位 源 时间</item>…<missing label="…">未找到</missing></raw_data>。块外不写一个字。</output_format>
```

## 4. 推理层框架（LLM 分析师协议）

`<thinking>` 里按序（实力与真金为先，赔率为辅）：
1. **引用原始证据**：逐场摘 Poly 1X2/让球/大小球 + KG 实力/伤停/战术 + 体彩赔率。对原文推。
2. **定剧本（不看体彩赔率）**：KG实力 + Poly真金 + 因果 → 谁赢/赢多少/节奏。
3. **估冷门**：每场冷门概率+触发条件，标最可能翻车那场（防守票目标）。
4. **赔率校验**：每候选并列 `命中概率` vs `盈亏平衡=1/赔率`，算 `edge`。命中<盈亏平衡=**−EV，老实标，禁"胜率高"伪装**。
5. **全工具选型**：剧本→体彩玩法；同剧本选胜率×盈亏比更优表达。
6. **分层组合（按§1.8 A/B/C/D 45/30/15/10）**：A层平胜负/不败背骨（45%）+ B层剧情爆发串（30%，≥2盘口同剧情）+ C层比分单关（15%，prob>5%且盘口一致）+ D层比分串高爆（10%，≥5×预算返奖）。倍投打预算；无edge的层→¥0。
7. **裁决**：给确定买单，每注标命中概率+盈亏比+层角色+中奖条件（一句话）。**无真edge→PASS+小A层，不强满买。** §1.10四过滤器须全过才出票。

## 5. 输出契约

- **格式走§1.10五步模板**（①盘口解构 ②剧情 ③资金分配 ④分层出票清单 ⑤审计7问）。
- 出票表：层(A/B/C/D) | 比赛 | 玩法 | 选择 | 体彩赔率 | P(命中) | edge | 金额 | 命中回报 | 中奖条件。
- 逐注写理由（实力+真金+因果→剧本→玩法选择）。
- 总投入=预算范围 | 最好/最坏情形 | 全场无+EV时明说"付水位买剧本"。
- **the out**：数据缺/锁盘→跳过该场，不整体劝退。
